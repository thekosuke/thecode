Otter.ai: Custom Icon Library (2021)

The past design contained inconsistencies in size and border width, which created technical difficulties when implementing the icons in the UI. For instance, each icon appeared to be in different proportions when they are all in 24px.

Even though the icons are in outline style, they lacked cohesion due to the lack of distinct characteristics that should exist among all the icons. From a functional standpoint, the icons were either too thin and hard to see when placed on a dark background, had inconsistent proportions that appeared like they're in different sizes, or inconsistent specifications in the Figma file that required custom adjustments.

Before: the previous icons were inconsistent in size and style

The existing icons were simple, modern, friendly, and sometimes quirky. Each icon was reduced to its minimal form, expressing essential characteristics. I aimed to adopt these principles while incorporating the functional aspect to it.

While adopting the latest design language, the new icons must respect practicality in which considers legibility and implementation. Given the existing brand and problems, the goal was to design a icon library that is unique to Otter by being practical, intuitive, and whimsical.

The visual concept was to figuratively Substract and Underline — as a general room of thumb, remove an element from the conventional icon to achieve a minimal effect while creating a visual element of recognition.

After: consistent proportions with the Otter brand injected

For example, the ‘Delete’ icon replaces the vertical lines inside the trash can with a circle that is reminiscient of the Otter ‘O’. If an icon has a smaller icon in the corner, the smaller icon should be filled to maximize legibility and add contrast to the outline form.

The icons were created in a 24px by 24px grid with 2px padding to create a consistent breathing space for each icon. I set the stroke weight to 1.75px to create icons that can easily be seen on a dark background, but also to appear round and approachable in a small form. The round caps and joints supported this notion of a friendly design language. All icons were converted into a flat vector that scales proportionately when set to different sizes.

Macro: Design System (2020)

As rough product concepts began to take shape, it was vital for Macro to implement a design system that helped the team design and build features quickly. This was for my own sanity as the sole designer, but also to enable the engineering team to efficiently ship features while refining the UI.

As an early-stage B2B software, Macro was committed to designing and building the product in a scalable and flexible way. This meant that the components had to be reusable in various screens to reduce engineering effort while saving space for future features. As the one designing this system, I always considered these two points while foreseeing how the product will evolve over time.
